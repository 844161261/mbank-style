### git基于分支的开发原则:
1. 更新远程仓库数据到本地仓库
1. 从本地仓库的主干`master`新建分支, 分支以此次修改要解决的问题/修复的bug命名
2. 在分支上做开发和测试
3. 没有问题后将分支合并到本地仓库主干(master)
4. push本地仓库的主干到远程仓库的主干
